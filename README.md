# PantryChef

PantryChef helps you turn whats in your pantry into delicious meals. Built with Flutter and Firebase, it stores your pantry items and recipes in Firestore, suggests recipes you can cook now, and makes it easy to add, edit, or remove recipes and ingredients.

**Highlights**
- Simple pantry-driven recipe recommendations ("Cook Now" and "Almost")
- Firestore-backed persistence with anonymous auth
- Add / edit / delete recipes and pantry items
- Search and category filtering for recipes
- Lightweight, production-ready Flutter app with a release APK included

**Quick Demo**
- Launch the app and add ingredients in the "Pantry" tab.
- The app shows "Cook Now" matches and "Almost" suggestions.
- Go to the "Recipes" tab to view, add, edit or delete recipes.

**Features**
- **Persistence:** Uses Cloud Firestore to save `users`, `recipes`, and `pantryItems`.
- **Seeding:** On first launch the app seeds a set of sample recipes and pantry items.
- **Search & Filter:** Find recipes by name or ingredients and filter by category.
- **Smooth UI:** Animated pantry chips, dismissible recipes, and polished dialogs.

**Files of Interest**
- `lib/main.dart` — app entry, UI, and interactions
- `lib/firestore_service.dart` — Firestore data layer (init, seed, CRUD)
- `lib/models.dart` — `Recipe`, `PantryItem`, and `UserProfile` models
- `lib/data.dart` — seed recipes and pantry items
- `lib/firebase_options.dart` — generated Firebase configuration

**Requirements**
- Flutter (recommended stable channel) — https://flutter.dev
- Dart SDK (bundled with Flutter)
- Android SDK (for Android builds)
- Firebase project (Firestore enabled) — see Firebase Setup below

**Firebase Setup (one-time)**
1. Create a Firebase project at https://console.firebase.google.com.
2. Enable Cloud Firestore in the console (start in test mode while developing).
3. (Optional) Register Android / iOS apps and download `google-services.json` / `GoogleService-Info.plist` if you prefer manual config.
4. The project already contains `lib/firebase_options.dart` generated by the FlutterFire CLI; if you re-run `flutterfire configure` it will update that file for your Firebase project.

**Run locally**
```bash
# Get dependencies
flutter pub get

# Run on connected device or emulator
flutter run
```

**Build release APK**
```bash
# Build a release APK
flutter build apk --release

# The generated APK will be at:
# build/app/outputs/flutter-apk/app-release.apk
```

**Build Android App Bundle (Play Store)**
```bash
flutter build appbundle --release
# Output: build/app/outputs/bundle/release/app-release.aab
```

**Notes on Signing**
- The current debug signing is used for convenience. For Play Store uploads, configure a proper signing key in `android/app/build.gradle.kts` and secure your keystore.

**Troubleshooting**
- Gradle plugin mismatch: If you hit a plugin version conflict, ensure `android/build.gradle.kts` and `android/settings.gradle.kts` reference the same `com.google.gms:google-services` plugin version.
- Firebase errors: Confirm `lib/firebase_options.dart` matches your Firebase project or re-run `flutterfire configure`.
- Missing Flutter on PATH: Run the Flutter binary directly, e.g. `C:\src\flutter\bin\flutter.bat analyze` on Windows.

**Developer Notes**
- Recipes are seeded on first launch by `FirestoreService._seedRecipesIfNeeded()` and saved under the `recipes` collection.
- Pantry items include a `userId` field and are stored in `pantryItems` with per-user isolation.
- To extend the app: add image fields to `Recipe`, create categories collection in Firestore, or add user authentication flows.
